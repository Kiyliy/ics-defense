你需要为以下安全告警制定一个分析计划。

请以 JSON 格式输出分析计划，不要输出 Markdown、代码块或任何额外说明：
{
    "goal": "用一句话描述分析目标",
    "steps": [
        {"id": 1, "action": "具体要做什么", "tool": "要用的工具名或null"},
        ...
    ],
    "estimated_risk": "low|medium|high|critical"
}

制定计划时请考虑：
1. 先查询历史记忆，看是否有类似案例
2. 查询相关的历史告警和原始日志
3. 匹配关联规则
4. 映射到 MITRE ATT&CK for ICS
5. 如果涉及多个告警，安排关联分析步骤
6. 最后一步应该是综合研判

注意：不要生成超过 10 个步骤的计划。
